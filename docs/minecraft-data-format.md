# minecraft-data output — field reference

This documents every JSON file emitted under `output/minecraft-data/<version>/`. Each Bedrock
version directory contains the **same 12 files**. One dedicated page per file lists every property,
its meaning, and the upstream source the value comes from.

## Files

| File                        | Shape  | What it is                                                 | Doc                                                                 |
| --------------------------- | ------ | ---------------------------------------------------------- | ------------------------------------------------------------------- |
| `attributes.json`           | array  | Player/entity attribute defaults & ranges                  | [attributes.md](./minecraft-data/attributes.md)                     |
| `biomes.json`               | array  | Biomes (bedrock ids enriched with java fields)             | [biomes.md](./minecraft-data/biomes.md)                             |
| `blocks.json`               | array  | Blocks (bedrock palette enriched with java fields)         | [blocks.md](./minecraft-data/blocks.md)                             |
| `blockStates.json`          | array  | Raw bedrock block-state palette (runtime-id order)         | [blockStates.md](./minecraft-data/blockStates.md)                   |
| `blockCollisionShapes.json` | object | Per-block collision shape indices + shape table            | [blockCollisionShapes.md](./minecraft-data/blockCollisionShapes.md) |
| `blocksJ2B.json`            | object | Java block-state string → Bedrock block-state string       | [blocksJ2B.md](./minecraft-data/blocksJ2B.md)                       |
| `blocksB2J.json`            | object | Bedrock block-state string → Java block-state string       | [blocksB2J.md](./minecraft-data/blocksB2J.md)                       |
| `items.json`                | array  | Items (bedrock registry enriched with java fields)         | [items.md](./minecraft-data/items.md)                               |
| `entities.json`             | array  | Entities (bedrock idlist + curated hitboxes + java fields) | [entities.md](./minecraft-data/entities.md)                         |
| `recipes.json`              | object | Crafting/smelting recipes (by recipe id)                   | [recipes.md](./minecraft-data/recipes.md)                           |
| `language.json`             | object | en_US localization key → string                            | [language.md](./minecraft-data/language.md)                         |
| `steve.json`                | object | Default player skin capture                                | [steve.md](./minecraft-data/steve.md)                               |

## How the data is produced

Each Bedrock version is generated by a worker thread (`src/generateData.ts` →
`src/extractWorker.ts` → `src/extract.ts`). The generators run **in this order**, because later ones
read the intermediate maps the earlier ones write:

1. `blockMap.ts` → `blockStates.json`, `blocksJ2B.json`, `blocksB2J.json`
2. `blocks.ts` → `blocks.json`
3. `collision.ts` → `blockCollisionShapes.json`
4. `itemMap.ts` → (intermediate item maps)
5. `items.ts` → `items.json`
6. `biomeMap.ts` → (intermediate biome maps)
7. `biomes.ts` → `biomes.json`
8. `entities.ts` → `entities.json`
9. `recipe.ts` → `recipes.json`
10. `language.ts` → `language.json`
11. `attributes.ts` → `attributes.json`
12. `steve.ts` → `steve.json`

Every Bedrock version is pinned to a Java version (and optionally a separate `javaItemsVersion`) plus
Geyser commit hashes in `src/versions.ts`. Java-derived fields come from those pins.

## Upstream data sources

See **[external-data-sources.md](../external-data-sources.md)** for the full catalog (origin,
pinning, fetch code, and a source→output matrix). Quick reference — provenance keywords used on each
page map to these origins:

| Source label                  | Origin                                                 | Files (in `data/…`)                                                                                                                                   | Downloaded by                   |
| ----------------------------- | ------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------- |
| **Bedrock packet capture**    | Live BDS connected via `bedrock-protocol`              | `bedrock/<ver>/item_registry.json`, `crafting_data.json`, `available_entity_identifiers.json`, `biome_definition_list.json`, `player_attributes.json` | `src/server/packet-exporter.ts` |
| **bedrock-data**              | `mc-zuri/bedrock-data` repo (keyed by `serverVersion`) | `bedrock/<ver>/block_palette.nbt`, `block-state-shapes.nbt`                                                                                           | `src/bedrock-data.ts`           |
| **Geyser mappings**           | `GeyserMC/mappings` @ pinned commit                    | `bedrock/<ver>/items_mappings.json`, `generator_blocks_v1.json`, `collision.json` / `collisions.nbt`                                                  | `src/geyser-mappings.ts`        |
| **Geyser mappings-generator** | `GeyserMC/mappings-generator` @ pinned `mg` commit     | `bedrock/<ver>/generator_blocks_v2.json`                                                                                                              | `src/geyser-mappings.ts`        |
| **Java minecraft-data**       | `mc-zuri/node-minecraft-data` @ `bedrock-v2`           | `java/<dir>/{biomes,blocks,entities,items}.json`                                                                                                      | `src/java-data.ts`              |
| **PyMCTranslate**             | `gentlegiantJGC/PyMCTranslate` @ pinned commit         | `pymctranslate/{java_26_2,bedrock_26_30}.json`                                                                                                        | `src/pymctranslate.ts`          |
| **Vanilla resource pack**     | Shipped inside each BDS download                       | `servers/<ver>/resource_packs/vanilla/texts/en_US.lang`                                                                                               | server download step            |

## Coverage

51 versions, `1.16.201` → `1.26.30` (see `src/versions.ts`). Format breakpoints (where a file's
shape or contents materially change between versions) are called out in a **Version breakpoints**
section on each page.
